179 // The shifter structure consists of a 33-bit shifter: 32-bit operand + 1 bit extension for
180 // arithmetic shifts and one-shift support.
181 // Rotations and funnel shifts are implemented as multi-cycle instructions.
182 // The shifter is also used for single-bit instructions and bit-field place as detailed below.
183 //
184 // Standard Shifts
185 // ===============
186 // For standard shift instructions, the direction of the shift is to the right by default. For
187 // left shifts, the signal shift_left signal is set. If so, the operand is initially reversed,
188 // shifted to the right by the specified amount and shifted back again. For arithmetic- and
189 // one-shifts the 33rd bit of the shifter operand can is set accordingly.
190 //
191 // Multicycle Shifts
192 // =================
193 //
194 // Rotation
195 // --------
196 // For rotations, the operand signals operand_a_i and operand_b_i are kept constant to rs1 and
197 // rs2 respectively.
198 //
199 // Rotation pseudocode:
200 // shift_amt = rs2 & 31;
201 // multicycle_result = (rs1 >> shift_amt) | (rs1 << (32 - shift_amt));
202 // ^-- cycle 0 -----^ ^-- cycle 1 --------------^
203 //
204 // Funnel Shifts
205 // -------------
206 // For funnel shifs, operand_a_i is tied to rs1 in the first cycle and rs3 in the
207 // second cycle. operand_b_i is always tied to rs2. The order of applying the shift amount or
208 // its complement is determined by bit [5] of shift_amt.
209 //
210 // Funnel shift Pseudocode: (fsl)
211 // shift_amt = rs2 & 63;
212 // shift_amt_compl = 32 - shift_amt[4:0]
213 // if (shift_amt >=33):
214 // multicycle_result = (rs1 >> shift_amt_compl[4:0]) | (rs3 << shift_amt[4:0]);
215 // ^-- cycle 0 ----------------^ ^-- cycle 1 ------------^
216 // else if (shift_amt <= 31 && shift_amt > 0):
217 // multicycle_result = (rs1 << shift_amt[4:0]) | (rs3 >> shift_amt_compl[4:0]);
218 // ^-- cycle 0 ----------^ ^-- cycle 1 -------------------^
219 // For shift_amt == 0, 32, both shift_amt[4:0] and shift_amt_compl[4:0] == '0.
220 // these cases need to be handled separately outside the shifting structure:
221 // else if (shift_amt == 32):
222 // multicycle_result = rs3
223 // else if (shift_amt == 0):
224 // multicycle_result = rs1.
225 //
226 // Single-Bit Instructions
227 // =======================
228 // Single bit instructions operate on bit operand_b_i[4:0] of operand_a_i.
229
230 // The operations bset, bclr and binv are implemented by generation of a bit-mask using the
231 // shifter structure. This is done by left-shifting the operand 32'h1 by the required amount.
232 // The signal shift_sbmode multiplexes the shifter input and sets the signal shift_left.
233 // Further processing is taken care of by a separate structure.
234 //
235 // For bext, the bit defined by operand_b_i[4:0] is to be returned. This is done by simply
236 // shifting operand_a_i to the right by the required amount and returning bit [0] of the result.
237 //
238 // Bit-Field Place
239 // ===============
240 // The shifter structure is shared to compute bfp_mask << bfp_off.

466 // The parallel prefix counter is of the structure of a Brent-Kung Adder. In the first
467 // log2(width) stages, the sum of the n preceding bit lines is computed for the bit lines at
468 // positions 2**n-1 (power-of-two positions) where n denotes the current stage.
469 // In stage n=log2(width), the count for position width-1 (the MSB) is finished.
470 // For the intermediate values, an inverse adder tree then computes the bit counts for the bit
471 // lines at positions
472 // m = 2**(n-1) + i*2**(n-2), where i = [1 ... width / 2**(n-1)-1] and n = [log2(width) ... 2].
473 // Thus, at every subsequent stage the result of two previously unconnected sub-trees is
474 // summed, starting at the node summing bits [width/2-1 : 0] and [3*width/4-1: width/2]
475 // and moving to iteratively sum up all the sub-trees.
476 // The inverse adder tree thus features log2(width) - 1 stages the first of these stages is a
477 // single addition at position 3*width/4 - 1. It does not interfere with the last
478 // stage of the primary adder tree. These stages can thus be folded together, resulting in a
479 // total of 2*log2(width)-2 stages.
480 // For more details refer to R. Brent, H. T. Kung, "A Regular Layout for Parallel Adders",
481 // (1982).
482 // For a bitline at position p, only bits
483 // bitcnt_partial[max(i, such that p % log2(i) == 0)-1 : 0] are needed for generation of the
484 // butterfly network control signals. The adders in the intermediate value adder tree thus need
485 // not be full 5-bit adders. We leave the optimization to the synthesis tools.
486 //
487 // Consider the following 8-bit example for illustraton.
488 //
489 // let bitcnt_bits = 8'babcdefgh.
490 //
491 // a b c d e f g h
492 // | /: | /: | /: | /:
493 // |/ : |/ : |/ : |/ :
494 // stage 1: + : + : + : + :
495 // | : /: : | : /: :
496 // |,--+ : : |,--+ : :
497 // stage 2: + : : : + : : :
498 // | : | : /: : : :
499 // |,-----,--+ : : : : ^-primary adder tree
500 // stage 3: + : + : : : : : -------------------------
501 // : | /| /| /| /| /| : ,-intermediate adder tree
502 // : |/ |/ |/ |/ |/ : :
503 // stage 4 : + + + + + : :
504 // : : : : : : : :
505 // bitcnt_partial[i] 7 6 5 4 3 2 1 0
506
507 always_comb begin
508 bitcnt_partial = '{default: '0};
509 // stage 1
510 for (int unsigned i = 1; i < 32; i += 2) begin
511 bitcnt_partial[i] = {5'h0, bitcnt_bits[i]} + {5'h0, bitcnt_bits[i-1]};
512 end
513 // stage 2
514 for (int unsigned i = 3; i < 32; i += 4) begin
515 bitcnt_partial[i] = bitcnt_partial[i-2] + bitcnt_partial[i];
516 end
517 // stage 3
518 for (int unsigned i = 7; i < 32; i += 8) begin
519 bitcnt_partial[i] = bitcnt_partial[i-4] + bitcnt_partial[i];
520 end
521 // stage 4
522 for (int unsigned i = 15; i < 32; i += 16) begin
523 bitcnt_partial[i] = bitcnt_partial[i-8] + bitcnt_partial[i];
524 end
525 // stage 5
526 bitcnt_partial[31] = bitcnt_partial[15] + bitcnt_partial[31];
527 // ^- primary adder tree
528 // -------------------------------
529 // ,-intermediate value adder tree
530 bitcnt_partial[23] = bitcnt_partial[15] + bitcnt_partial[23];
531
532 // stage 6
533 for (int unsigned i = 11; i < 32; i += 8) begin
534 bitcnt_partial[i] = bitcnt_partial[i-4] + bitcnt_partial[i];
535 end
536
537 // stage 7
538 for (int unsigned i = 5; i < 32; i += 4) begin
539 bitcnt_partial[i] = bitcnt_partial[i-2] + bitcnt_partial[i];
540 end
541 // stage 8
542 bitcnt_partial[0] = {5'h0, bitcnt_bits[0]};
543 for (int unsigned i = 2; i < 32; i += 2) begin
544 bitcnt_partial[i] = bitcnt_partial[i-1] + {5'h0, bitcnt_bits[i]};
545 end
546 end

735 // The crossbar permutation instructions xperm.[nbh] (Zbp) can be implemented using 8

822 ///////////////////////////////////////////////////
823 // Carry-less Multiply + Cyclic Redundancy Check //
824 ///////////////////////////////////////////////////
825
826 // Carry-less multiplication can be understood as multiplication based on
827 // the addition interpreted as the bit-wise xor operation.
828 //
829 // Example: 1101 X 1011 = 1111111:
830 //
831 // 1011 X 1101
832 // -----------
833 // 1101
834 // xor 1101
835 // ---------
836 // 10111
837 // xor 0000
838 // ----------
839 // 010111
840 // xor 1101
841 // -----------
842 // 1111111
843 //
844 // Architectural details:
845 // A 32 x 32-bit array
846 // [ operand_b[i] ? (operand_a << i) : '0 for i in 0 ... 31 ]
847 // is generated. The entries of the array are pairwise 'xor-ed'
848 // together in a 5-stage binary tree.
849 //
850 //
851 // Cyclic Redundancy Check:
852 //
853 // CRC-32 (CRC-32/ISO-HDLC) and CRC-32C (CRC-32/ISCSI) are directly implemented. For
854 // documentation of the crc configuration (crc-polynomials, initialization, reflection, etc.)
855 // see http://reveng.sourceforge.net/crc-catalogue/all.htm
856 // A useful guide to crc arithmetic and algorithms is given here:
857 // http://www.piclist.com/techref/method/math/crcguide.html.
858 //
859 // The CRC operation solves the following equation using binary polynomial arithmetic:
860 //
861 // rev(rd)(x) = rev(rs1)(x) * x**n mod {1, P}(x)
862 //
863 // where P denotes lower 32 bits of the corresponding CRC polynomial, rev(a) the bit reversal
864 // of a, n = 8,16, or 32 for .b, .h, .w -variants. {a, b} denotes bit concatenation.
865 //
866 // Using barret reduction, one can show that
867 //
868 // M(x) mod P(x) = R(x) =
869 // (M(x) * x**n) & {deg(P(x)'{1'b1}}) ^ (M(x) x**-(deg(P(x) - n)) cx mu(x) cx P(x),
870 //
871 // Where mu(x) = polydiv(x**64, {1,P}) & 0xffffffff. Here, 'cx' refers to carry-less
872 // multiplication. Substituting rev(rd)(x) for R(x) and rev(rs1)(x) for M(x) and solving for
873 // rd(x) with P(x) a crc32 polynomial (deg(P(x)) = 32), we get
874 //
875 // rd = rev( (rev(rs1) << n) ^ ((rev(rs1) >> (32-n)) cx mu cx P)
876 // = (rs1 >> n) ^ rev(rev( (rs1 << (32-n)) cx rev(mu)) cx P)
877 // ^-- cycle 0--------------------^
878 // ^- cycle 1 -------------------------------------------^
879 //
880 // In the last step we used the fact that carry-less multiplication is bit-order agnostic:
881 // rev(a cx b) = rev(a) cx rev(b).
882
883 logic clmul_rmode;
884 logic clmul_hmode;

998 ///////////////
999 // Butterfly //
1000 ///////////////
1001
1002 // The butterfly / inverse butterfly network executing bcompress/bdecompress (zbe)
1003 // instructions. For bdecompress, the control bits mask of a local left region is generated
1004 // by the inverse of a n-bit left rotate and complement upon wrap (LROTC) operation by the
1005 // number of ones in the deposit bitmask to the right of the segment. n hereby denotes the
1006 // width of the according segment. The bitmask for a pertaining local right region is equal
1007 // to the corresponding local left region. Bcompress uses an analogue inverse process.
1008 // Consider the following 8-bit example. For details, see Hilewitz et al. "Fast Bit Gather,
1009 // Bit Scatter and Bit Permuation Instructions for Commodity Microprocessors", (2008).
1010 //
1011 // The bcompress/bdecompress instructions are completed in 2 cycles. In the first cycle, the
1012 // control bitmask is prepared by executing the parallel prefix bit count. In the second
1013 // cycle, the bit swapping is executed according to the control masks.
1014
1015 // 8-bit example: (Hilewitz et al.)
1016 // Consider the instruction bdecompress operand_a_i deposit_mask
1017 // Let operand_a_i = 8'babcd_efgh
1018 // deposit_mask = 8'b1010_1101
1019 //
1020 // control bitmask for stage 1:
1021 // - number of ones in the right half of the deposit bitmask: 3
1022 // - width of the segment: 4
1023 // - control bitmask = ~LROTC(4'b0, 3)[3:0] = 4'b1000
1024 //
1025 // control bitmask: c3 c2 c1 c0 c3 c2 c1 c0
1026 // 1 0 0 0 1 0 0 0
1027 // <- L -----> <- R ----->
1028 // operand_a_i a b c d e f g h
1029 // :\ | | | /: | | |
1030 // : +|---|--|-+ : | | |
1031 // :/ | | | \: | | |
1032 // stage 1 e b c d a f g h
1033 // <L-> <R-> <L-> <R->
1034 // control bitmask: c3 c2 c3 c2 c1 c0 c1 c0
1035 // 1 1 1 1 1 0 1 0
1036 // :\ :\ /: /: :\ | /: |
1037 // : +:-+-:+ : : +|-+ : |
1038 // :/ :/ \: \: :/ | \: |
1039 // stage 2 c d e b g f a h
1040 // L R L R L R L R
1041 // control bitmask: c3 c3 c2 c2 c1 c1 c0 c0
1042 // 1 1 0 0 1 1 0 0
1043 // :\/: | | :\/: | |
1044 // : : | | : : | |
1045 // :/\: | | :/\: | |
1046 // stage 3 d c e b f g a h
1047 // & deposit bitmask: 1 0 1 0 1 1 0 1
1048 // result: d 0 e 0 f g 0 h
1049
1050 logic [ 5:0] bitcnt_partial_q [32];
1051
1052 // first cycle
1053 // Store partial bitcnts
1054 for (genvar i = 0; i < 32; i++) begin : gen_bitcnt_reg_in_lsb
1055 assign bitcnt_partial_lsb_d[i] = bitcnt_partial[i][0];
1056 end

1060 end

1062 for (genvar i = 0; i < 8; i++) begin : gen_bitcnt_reg_in_b2
1063 assign bitcnt_partial_msb_d[16+i] = bitcnt_partial[4*i+3][2];
1064 end
1065
1066 for (genvar i = 0; i < 4; i++) begin : gen_bitcnt_reg_in_b3
1067 assign bitcnt_partial_msb_d[24+i] = bitcnt_partial[8*i+7][3];
1068 end
